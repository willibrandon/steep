Tables View with Size Trend Sparklines
=======================================

Overview:
- Add "Trend" column showing table size changes over 24h
- Uses existing sparkline.go component (RenderUnicodeSparkline)
- Column positioned between Cache and Vacuum columns
- Hidden when terminal width < 100 characters

Sparkline Patterns:
- Growing table:   ▁▂▃▄▅▆▇█  with ↑ indicator (green)
- Stable table:    ▄▄▄▄▄▄▄▄  with → indicator (dim)
- Shrinking table: █▇▆▅▄▃▂▁  with ↓ indicator (yellow)

Column Details:
- Width: 10 characters (8 for sparkline + 1 for trend arrow + 1 padding)
- Data source: SQLite metrics_history table with key="table_size:<schema>.<table>"
- Time window: 24 hours
- Sparkline width: 8 characters (compact for table cells)

Data Flow:
1. TablesView fetches table data (existing flow)
2. After data refresh, record current sizes to MetricsStore with keyed metric
3. Query MetricsStore for each visible table's size history
4. Render sparkline using RenderUnicodeSparkline from sparkline.go
5. Add trend indicator using GetTrend()

Implementation Notes:
- Use NEW keyed functions in MetricsStore (don't modify existing SaveDataPoint/GetHistory)
- Add SaveKeyedDataPoint(), GetKeyedHistory() as separate functions
- Store with metric_name="table_size", key="<schema>.<table>"
- Cache sparkline data in TablesView to avoid repeated queries during render
- Refresh cache when data refreshes (on RefreshTablesMsg)
